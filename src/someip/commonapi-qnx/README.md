# CommonAPI-QNX

Cross-compile **vsomeip 3.5.5**, **CommonAPI Core Runtime 3.2.4**,
**CommonAPI SOME/IP Runtime 3.2.4**, and **Boost 1.84.0** for QNX using
the QNX `qcc`/`q++` compiler.

## Directory layout

```
commonapi-qnx/
├── Makefile              # Top-level: make / make download / make patch / make build / make clean
├── scripts/
│   ├── env.sh            # QNX env + all exported config macros (source me)
│   ├── download.sh       # Downloads all third-party sources
│   ├── patch.sh          # Applies QNX-specific patches to vsomeip
│   └── build.sh          # Cross-compiles libraries for QNX
│
├── build-rpi/            # ← generated by build.sh (rpi profile)
│   ├── lib/              #   .so libraries
│   ├── include/          #   public headers
│   ├── generators/       #   unzipped CommonAPI code generators
│   ├── cmake-build/      #   intermediate CMake build dirs (deletable)
│   ├── toolchain.cmake   #   QNX CMake toolchain file
│   ├── user-config.jam   #   Boost.Build config for qcc
│   ├── compat-include/   #   stub headers for missing QNX APIs
│   └── patches/          #   saved patch files
│
├── build-native/         # ← generated for native x86_64 (set QNX_TARGET_PROFILE=native)
│
├── third_party/          # ← created by download.sh
│   ├── vsomeip/                      # git tag 3.5.5
│   ├── capicxx-core-runtime/         # git tag 3.2.4
│   ├── capicxx-someip-runtime/       # git tag 3.2.4
│   ├── boost_1_84_0/                # release tarball
│   ├── commonapi_core_generator.zip  # generator 3.2.15
│   └── commonapi_someip_generator.zip# generator 3.2.15
│
└── README.md
```

## Configuration macros

All configuration is exported by `scripts/env.sh`. Override before running:

| Macro | Default | Description |
|-------|---------|-------------|
| `QNX_SDP_PATH` | auto-detect | Path to QNX SDP dir (contains `qnxsdp-env.sh`) |
| `QNX_TARGET_PROFILE` | `rpi` | `rpi` (aarch64le) or `native` (x86_64) |
| `OUTPUT_DIR` | `build-<profile>` | Where build artifacts go (forced absolute) |
| `BUILD_TYPE` | `Release` | CMake build type |
| `JOBS` | `nproc` | Parallel build jobs |
| `VSOMEIP_VERSION` | `3.5.5` | vsomeip git tag |
| `COMMONAPI_CORE_RUNTIME_VERSION` | `3.2.4` | Core runtime git tag |
| `COMMONAPI_SOMEIP_RUNTIME_VERSION` | `3.2.4` | SOME/IP runtime git tag |
| `COMMONAPI_GENERATOR_VERSION` | `3.2.15` | Generator release tag |
| `BOOST_VERSION` | `1.84.0` | Boost release version |

## Quick build (Makefile)

The Makefile has smart skip logic — it won't re-download or rebuild unless
`FORCE_REBUILD=1` is set:

```bash
cd commonapi-qnx

# Source QNX first (or set QNX_SDP_PATH)
source ../../qnx800/qnxsdp-env.sh

# One-command build (download + patch + build)
make

# Or step by step:
make download      # Download sources (skips if already done)
make patch         # Apply QNX patches (runs download first)
make build         # Build all libraries (runs patch first)
make clean         # Remove build-rpi/ and third_party/
```

For native x86_64 build:
```bash
export QNX_TARGET_PROFILE=native
make build
```

Rebuild from scratch:
```bash
FORCE_REBUILD=1 make build
```

## How to build the libraries (manual steps)

```bash
cd commonapi-qnx

# 1. Point to your QNX SDP installation
export QNX_SDP_PATH=/path/to/qnx/sdp

# 2. Set the output directory (optional, default: build-<profile>)
export OUTPUT_DIR=build-rpi

# 3. Download all sources at the versions in env.sh
bash scripts/download.sh

# 4. Apply QNX-specific patches to vsomeip (required for QNX networking)
bash scripts/patch.sh

# 5. Build all libraries (sources env.sh internally)
bash scripts/build.sh
```

Build for **native x86_64** instead of RPi:
```bash
export QNX_TARGET_PROFILE=native
bash scripts/build.sh
```

### Source env.sh interactively

You can also source `env.sh` to get all variables in your shell:

```bash
source scripts/env.sh
echo $LIBS_DIR    # → path to commonapi-qnx root
echo $OUTPUT_DIR  # → build output directory
```

`LIBS_DIR` is used by server/client CMakeLists to locate libraries and toolchain.

## Patch details

`patch.sh` applies a QNX-specific fix to vsomeip's `routing_manager_impl.cpp`:

- Sets `sd_route_set_ = true` at startup — no netlink connector on QNX to detect multicast routes
- Calls `init_pending_services()` after IP routing starts so locally-offered services get their TCP/UDP endpoints created

The patch is saved to `build-rpi/patches/vsomeip-qnx-pending-services.patch`.

## Build steps (inside build.sh)

1. **Boost 1.84.0** — builds system, thread, filesystem, log libraries with `b2`
2. **Unzip CommonAPI generators** 3.2.15 (core + someip, pre-built)
3. **vsomeip 3.5.5** — cross-compiled via CMake with QNX toolchain
4. **CommonAPI Core Runtime 3.2.4** — cross-compiled via CMake
5. **CommonAPI SOME/IP Runtime 3.2.4** — cross-compiled via CMake (links vsomeip + core)

### Compat headers

`build.sh` generates stub headers for QNX API gaps in `${OUTPUT_DIR}/compat-include/`:

- `sys/sockmsg.h` — wraps `sys/iomsg.h` + `sys/neutrino.h`
- `sys/eventfd.h` — provides `eventfd()` stub (always returns -1)

The toolchain file (`toolchain.cmake`) adds `-I${COMPAT_INCLUDE}` to all compiler flags.

After `build.sh` completes, copy these to the QNX target:

### Libraries (13 files)
```
build-rpi/lib/libboost_atomic.so

build-rpi/lib/libboost_log.so
build-rpi/lib/libboost_log_setup.so
build-rpi/lib/libboost_system.so
build-rpi/lib/libboost_thread.so

build-rpi/lib/libvsomeip3-cfg.so
build-rpi/lib/libvsomeip3-e2e.so
build-rpi/lib/libvsomeip3-sd.so
```

### Toolchain + generators (for building apps on this host)
```
build-rpi/toolchain.cmake     # used by server/client CMakeLists
build-rpi/generators/          # used by server/client at configure time
```

### On the target, set the library path:
```sh
export LD_LIBRARY_PATH=/path/to/libs:$LD_LIBRARY_PATH
```

The server/client executables are built separately from `commonapi_client_server/`
— see that project's README for instructions.
